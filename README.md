# 💎 Durcoin Wallet

**Лёгкий некастодиальный веб-кошелёк для блокчейна [Waves](https://waves.tech)**

Один PHP-файл. Без бэкенд-фреймворков. Без серверного хранения ключей. Работает как PWA.

## ✨ Возможности

- 🔐 **Некастодиальный** — seed-фраза и ключи **никогда** не покидают браузер
- 🔢 **PIN-код** — seed шифруется и хранится локально, разблокировка по PIN
- 📤 **Отправка** WAVES и любых токенов (Transfer v2)
- 🏷 **Алиасы** — создание и отправка на алиас вместо адреса
- ⛽ **Спонсируемая комиссия** — оплата комиссии токеном (sponsored fee)
- 📥 **Приём** — QR-код адреса + кнопка «Поделиться»
- 📷 **QR-сканер** — сканирование адреса камерой
- 📖 **Адресная книга** — локальные контакты
- 📜 **История** транзакций
- 🌗 **Тёмная/светлая** тема
- 🌍 **RU / EN** локализация
- 📱 **PWA** — устанавливается как приложение
- 🛡 **Прокси через сервер** — broadcast идёт через ваш PHP (обход блокировок ноды)

## 🚀 Быстрый старт

### Требования
- PHP **7.4+** с расширением **cURL** (рекомендуется) или `allow_url_fopen`
- Веб-сервер с **HTTPS** (обязательно для PWA, камеры и `crypto`)
- Доступ сервера к Waves-ноде (по умолчанию `nodes.wavesnodes.com`)

### Установка

```bash
git clone https://github.com/ВАШ_ЛОГИН/durcoin-wallet.git
cd durcoin-wallet
```

Залейте файлы на хостинг с PHP и HTTPS — и всё. **Сборка не нужна.**

Откройте `https://ваш-домен/` в браузере.

### Проверка доступа сервера к ноде
```
https://ваш-домен/?action=balance&address=3P_любой_адрес
```
Если вернулся JSON с балансом — сервер видит ноду, всё готово.

## ⚙️ Конфигурация

Параметры в начале `index.php`:

```php
$FORCE_TITLE   = '';                              // принудительный заголовок (пусто = из домена)
$BRAND_SUFFIX  = 'powered by Durcoin';            // подпись под логотипом
$WAVES_NODE    = 'https://nodes.wavesnodes.com';  // адрес ноды
$CHAIN_ID      = 'W';                             // 'W' = mainnet, 'T' = testnet
```

> 💡 Для тестов используйте `$CHAIN_ID = 'T'` и testnet-ноду `https://nodes-testnet.wavesnodes.com`.

## 🏗 Архитектура

```
Браузер (клиент)                  Ваш сервер (PHP)              Waves-нода
─────────────────                 ───────────────               ──────────
Генерация ключей    ──┐
Подпись транзакций    │  ✅ локально, ключи не уходят
Шифрование seed     ──┘

fetch ?action=balance ──────────► httpGet() ─────────────────► /addresses/balance
fetch ?action=broadcast (POST) ─► curl POST ─────────────────► /transactions/broadcast
fetch ?action=txinfo ───────────► httpGet() ─────────────────► /transactions/info
```

**Почему через сервер?** Если у пользователя заблокирован доступ к ноде (провайдер/регион), запросы всё равно проходят — их проксирует ваш сервер. Браузеру внешние домены не нужны (`connect-src 'self'`).

## 🔒 Безопасность

| Что | Как реализовано |
|---|---|
| Приватные ключи | Генерируются и используются **только в браузере** |
| Seed-фраза | Шифруется PIN-ом (XOR-поток на базе SHA-256), хранится в `localStorage` |
| Серверное хранение | **Отсутствует** — сервер только проксирует публичные запросы |
| CSP | Жёсткая политика, `connect-src 'self'` |
| Заголовки | `X-Frame-Options: DENY`, `Referrer-Policy: no-referrer` |
| Криптография | SHA-256 (чистый JS, самопроверка), blake2b, keccak256, Curve25519 |

> ⚠️ **Дисклеймер:** это open-source проект «как есть». Перед использованием с реальными средствами **проведите собственный аудит**. PIN-шифрование — базовая защита локального хранилища, не замена аппаратного кошелька.

## 🧪 Тестирование

1. Создайте новый кошелёк → **сохраните seed-фразу**
2. Переключитесь на **testnet** (`$CHAIN_ID='T'`) для безопасных тестов
3. Получите тестовые WAVES из [крана](https://faucet.wavesnodes.com)
4. Отправьте небольшую сумму на свой второй адрес
5. Проверьте появление в истории

> 💡 Первую реальную транзакцию делайте на **минимальной сумме**.

## 📦 Зависимости (вендорные, без CDN)

Все библиотеки лежат рядом — **никаких внешних CDN** (для приватности и работы офлайн):

| Файл | Назначение |
|---|---|
| `sha3.min.js` | keccak256 (адреса) |
| `axlsign.min.js` | подпись Curve25519 |
| `blakejs.min.js` | blake2b (ID транзакций) |
| `jsQR.min.js` | сканер QR |
